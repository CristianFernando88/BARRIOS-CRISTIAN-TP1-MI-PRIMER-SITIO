# Notas de Terminal - TP1

## Comandos usados

### 1. `pwd`
- **Qué hace:** Muestra la ruta completa del directorio actual.  
- **Situación:** Lo usé para saber en qué carpeta estaba ubicado y decidir si debía moverme hacia adelante o atrás en la estructura de directorios.  

### 2. `ls`
- **Qué hace:** Lista los archivos y carpetas dentro del directorio actual.  
- **Situación:** Lo usé para verificar qué archivos se habían creado y qué carpetas tenía disponibles.  

### 3. `cd`
- **Qué hace:** Cambia de directorio.  
- **Situación:** Lo usé para entrar en la carpeta principal donde guardé mi proyecto.  

### 4. `cd ..`
- **Qué hace:** Sube un nivel en la estructura de directorios.  
- **Situación:** Lo usé para volver atrás cuando estaba dentro de una carpeta equivocada.  

### 5. `mkdir tp1-mi-sitio`
- **Qué hace:** Crea una nueva carpeta.  
- **Situación:** Lo usé para crear la carpeta del TP1 y también la carpeta `img` para las imágenes.  

### 6. `touch index.html`
- **Qué hace:** Crea un archivo vacío.  
- **Situación:** Lo usé para crear los archivos `index.html`, `style.css`, `README.md` y `notas-terminal.md`.  

### 7. `cat README.md`
- **Qué hace:** Muestra el contenido de un archivo en la terminal.  
- **Situación:** Lo usé para revisar rápidamente el contenido del README sin abrir un editor.  

### 8. `which git`
- **Qué hace:** Muestra la ruta donde está instalado Git en el sistema.  
- **Situación:** Lo usé para documentar en el README la ubicación exacta de Git en mi PC.  

### 9. `code .`
- **Qué hace:** Abre Visual Studio Code en el directorio actual.  
- **Situación:** Lo usé para abrir todo el proyecto y editar los archivos de manera más cómoda.  

### 10. `clear`
- **Qué hace:** Limpia la pantalla de la terminal.  
- **Situación:** Lo usé para mantener la terminal ordenada y más fácil de leer mientras trabajaba.  

### 11. `git config --list`
- **Qué hace:** Ver los datos de configuracion del git.  
- **Situación:** Lo usé para mostrar user.name, user.email, core.editor e init.defaultbranch. .
